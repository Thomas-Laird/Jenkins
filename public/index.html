Welcome to my website. This is a work-in-progress, and will probably remain so for the forseeable future. The main aim of this site is to serve as content that I may use to deploy in my personal projects. I will be spending much more time experimenting with different technologies, e.g. pipelining or infrastructure as code tools. Any static content provided will hence have a low priority in my schedule.

From the above perspective even bothering to produce web content might seem superfluous and a waste of time. From a narrow perspective this might be so; I could always simply deploy prepackaged images or website templates, which available online in multitudes. The truth is not so, as directly controlling this content enables me to branch out and directly interface between different internet resources that I might run. This I have more whole architectures to work with; a much more useful and challenging operational experience. Besides, even simply creating this static content in an incremental fashion incentivises me to practise continous delivery of my content via some pipelining process. The ultimate goal would be that, the more complex this whole website infrastructure becomes, I will still need to maintain a way to spin the whole website up in an automated fashion.
